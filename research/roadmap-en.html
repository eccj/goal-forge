What it actually proves

Mechanism design, game theory, probability, and security lenses on the core promise. The result tightens the claim — and it's more honest than the old headline.

Proven (≈prob 1, correlation-independent)

  • In-place ledger tamper → verify cascades CHAIN BROKEN from the edit forward (reduces to SHA-256).
  • Fabricated/failing mechanical evidence → J1 re-runs by its own command & pastes git-diff; a shimmed test is caught.
  • Structural omission / bare "done, passed" → completion gate + <anti-accept> reject it.

Bluff (weak / probabilistic)

  • Semantic Goodhart gap — a passing-but-vacuous assertion survives; est. 13–45% per subtle deliverable, in-family-bounded.
  • Juror capture — the worker spawns its own jury; it can stage inputs so real jurors approve a curated view.
  • Weak prosecutor & compaction erosion of non-harness FORBIDDEN constraints.
Tightest honest claim
"A self-interested agent can't say done without proving it."
→ A non-strategic worker cannot cheaply over-claim a mechanically-decidable deliverable, and all fakery is tamper-evident to a later external re-runner. Tamper-evidence, not prevention; ex-post auditability, not autonomous credibility. This exactly validates the honest-ceiling ethos.

python vs Go vs Rust · 20 opus agents

Language decision

bash / python>Go>Rust
  • Core & trust path stay bash + shasum. The ledger's value is "the artifact you run is the source you read" — J2 recomputes the chain by hand. A compiled binary is un-auditable-without-a-toolchain and un-installable-by-default; it detonates the trust story. Go and Rust are both barred here.
  • Rust is worse than Go for this skill — no stdlib crypto (a sha2 crate = the exact dependency creep GUARDRAILS forbids), a 1–1.5 GB toolchain, slowest compile, least readable.
  • New explicit invariant: "no opaque compiled artifact in the completion-trust path; hand-auditable with universal built-ins."
  • Additive (not migration): add Go & Rust deliverable recipes so a user's Go/Rust project emits native evidence. If bash write-time fragility ever bites, the escape hatch is Go — never Rust — with bash retained as the readable reference.
Beyond the hardening pass

v1.9–2.0 backlog & the landscape

30 fresh opus agents · 3 groups · converged

v1.9 direction — "bind the seams." Three independent 10-agent groups (capability R&D · external research · red-team) arrived at one diagnosis from three angles: 1.8 secured the interior of a single chain, but every place trust must cross a boundary is still authored by the worker — the protocol's own adversary.

  • The live core hole (capability): the ledger chains from a literal GENESIS with no session/identity binding — so a resumed or compaction-amnesiac session reloads from disk and a full re-forge still prints CHAIN INTACT. J2's from-GENESIS recompute can't catch it; a re-forged chain is internally consistent. → session-seam braid to a git-content witness.
  • The research consensus: verification over evaluation — shrink the LLM-judgment surface to deterministic, ledger-anchored oracle checks; push the completion gate toward a harness Stop-hook, not the worker's own turn.
  • The red-team consensus: every 1.8 anchor is written/resolved/invoked by the worker — oracle-poisoning, near-free re-forge, jury-shopping. The fix is attestation: bind each anchor to something the worker can't quietly control, and ship an executable attack fixture that flips red pre-fix / green on HEAD.

All three keep the invariants: keyless · bash + shasum + git · zero external-API · SKILL ≤ 97. This is the first real structural answer to the honest ceiling 1.8 could not close — the worker convening its own jury.

Research-grounded backlog (top ideas)
  • Ambig-DSCompile-time framing lock — resolve/record the 2–3 plausible mission readings before compiling. Ambiguous runs misframe silently 39–63% of the time; the ledger is blind to it.
  • SAuditable completion gate — STOP_REASON grammar + coverage assertion. Cheapest, most mechanical win.
  • RHBVerifier-artifact + hash-pinning — freeze acceptance checks at D1; drift is a non-overridable S-finding.
